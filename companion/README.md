# `companion/` — interactive Neo4j-backed knowledge graph

The atlas itself is static — Markdown + SVG + YAML — and renders on
GitHub from anywhere with no infrastructure required. **This folder is
the optional interactive companion**: a Neo4j-backed app generated by
[`create-context-graph`][ccg] that lets you query the registry as a
graph (who-implements-which-component, model-agnosticism histograms,
substitution-path walks for pattern-fire scenarios).

The companion is opt-in. The atlas does not depend on it.

> **Status: v0.1.0 stub.** The domain YAML below describes the
> *intended* node and relationship shape. End-to-end companion build is
> the v0.3.0 release deliverable per `plan.md` and the `CHANGELOG.md`
> release ladder.

[ccg]: https://github.com/Benjamin-KY/create-context-graph

## What the companion gives you

| Use case | How it shows up |
|---|---|
| "Which governance projects implement the audit-log FSM?" | Cypher query over `Harness -[COVERS_COMPONENT]-> Component` edges. |
| "If `claude-agent-sdk` goes away, which adjacent entries cover the same surface?" | Traverse `Harness -[ADJACENT_TO]- Harness`, filter by overlapping `COVERS_COMPONENT` set. |
| "Top-N most model-agnostic entries per category" | `Harness` nodes ordered by `model_agnostic_score`, faceted by `category`. |
| "Sovereignty radial — origin × license × agnosticism" | Aggregation over `Harness` + `MAINTAINED_BY -> Maintainer`. |

These same questions are answerable by reading the YAML by hand — the
companion just lets you ask them at the speed of a query rather than
the speed of a grep.

## Prerequisites

- Docker (for the local Neo4j container)
- Python 3.11+ with this repo's dev environment installed
  (`pip install -e ".[dev]"`)
- `create-context-graph` installed:
  ```powershell
  pip install create-context-graph
  ```

## Quickstart

```powershell
# 1. Start a local Neo4j 5 container
make neo4j-local

# 2. Generate the domain YAML from the registry
make companion
# (writes companion/domain/open-harnesses.yaml — touches the
#  updated_at timestamp in v0.1.0; full generation lands in v0.3.0)

# 3. Scaffold the app
cd companion
create-context-graph ./app `
    --domain ./domain/open-harnesses.yaml `
    --framework pydanticai `
    --demo-data

# 4. Run it
cd app
# follow the create-context-graph project's own README from here
```

## Why a separate Neo4j companion at all?

The static atlas is the front door for everyone. The companion is the
**power-user surface** for catalog maintainers, deep-dive contributors,
and anyone running their own portability analysis (e.g., "what is my
exposure to a Fable / Mythos-style shock?"). Both share the same source
of truth — the YAML in `registry/`.

## What is *not* in the companion

- LLM-fabricated nodes or relationships. Fixtures are derived from the
  registry, full stop. The companion does not call an LLM to populate
  the graph.
- Hosted state. Everything runs on your machine; no telemetry by
  default. The companion inherits this from `create-context-graph`'s
  posture.
- Authentication. The companion ships with `neo4j/change-me-please`
  defaults for *local* development and is not intended for a shared /
  exposed deployment without re-configuring credentials and adding an
  auth layer in front.
