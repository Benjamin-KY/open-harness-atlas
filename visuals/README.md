# `visuals/` — design notes

All visuals here are released under **CC BY-SA 4.0** (`LICENSE-DOCS`), like
the rest of the catalog content. The accompanying code (`scripts/`) is
under Apache-2.0.

## Hand-authored

These are committed directly as SVG files and edited by hand. They are not
regenerated by CI.

| File | Source of truth | Notes |
|------|-----------------|-------|
| `taxonomy.svg` | This repo's six categories | Edit by hand when a category is added / renamed; mirror to `README.md` and `docs/taxonomy.md`. |
| `five-component-overlay.svg` | `registry/governance/*.yaml` `five_component_coverage` | Edit by hand whenever a governance entry's coverage values change; re-render `docs/five-component-overlay.md` to keep both in sync. |

## Data-driven (generated)

These are emitted by `scripts/build_visuals.py` from registry + sidecar
data, on every release and on every `make build`. They are committed so
GitHub renders them inline in the README without needing the script to be
re-run on view.

| File | Generator | Slated for |
|------|-----------|------------|
| `model-agnostic-spectrum.svg` | `build_visuals.py` from `model_agnostic_score` across all entries | v0.2.0 |
| `sovereignty-radial.svg` | `build_visuals.py` from license / origin / agnostic axes | v0.2.0 |
| `fable-mythos-pattern-fire.svg` | Hand-authored worked-example illustration | v0.4.0 |

## Palette (mirrors `harmless-harnesses` BRAND.md)

| Hex | Role |
|-----|------|
| `#1f3a5f` | Spine / harness / governance |
| `#28a745` | Deterministic logic / eval |
| `#d68910` | LLM / stochastic component / agent |
| `#7d3c98` | Data store / education / knowledge |
| `#c0392b` | Failure mode / adversarial / red-team |
| `#b2b8c0` | Inactive / out-of-scope (used for `none` cells) |

All visuals render on a `#ffffff` background and use system fonts so they
work in dark / light browser themes without baking in a font dependency.

## Why SVG over PNG

SVG is text-diffable, scales cleanly on retina / hi-DPI screens, can be
inspected from a screen reader (each SVG has `<title>` + `<desc>`), and
keeps the registry-to-visual provenance auditable — you can see the colour
of every cell in plain XML.
